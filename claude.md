# AI Usage Guidelines

## Purpose

AI tools were used to accelerate development while maintaining system correctness and structure.

## Usage

- Generated initial scaffolding and boilerplate
- Assisted in structuring backend and frontend components
- Suggested patterns for API design and state handling

## Validation

- All generated code was manually reviewed
- Business logic verified through 33 unit tests
- Edge cases tested explicitly

## Constraints

- No blind copy-paste of AI output
- Maintained modular architecture
- Ensured separation of concerns

## Outcome

AI-assisted development improved speed while preserving correctness and maintainability.
